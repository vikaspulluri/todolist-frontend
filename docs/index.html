Todolists App Events

(V 1.0.0)

setUser

This event should be emitted when the user is authenticated in client side. It should emit the following data.

{authToken: 'User Authorized Token', friendsList: ['123','345']}

Note: Passing friendsList in above object is optional, as we have separate emitter to do that

setFriends

This event should be emitted when the user friends data fetched successfully in cliet side. It should emit the following data.

[{firstName: 'Karthik', lastName: 'G', _id: '123445'}, {firstName: 'Noothana', lastName: 'P', _id: '344543'} ]

All frieds will be joined in current user room by emitting this event

getOnlineUsers

This event should be emitted when the user is authenticated in client side to get the all online users list. It won't emit any data and will listen for onlineUserList event to get the online users data.

sendFriendRequest

This event should be emitted when the user is sends a friend request to another user. It should emit the following data.

{requesterId: 'currentUserId', requesterName: 'currentUserName', receiverId: 'id of the receiver', receiverName: 'Name of the receiver', status: 'Pending'}

acceptFriendRequest

This event should be emitted when the user accepts a friend request received from another user. It should emit the following data.

{requesterId: 'requesterUserId', requesterName: 'requesterUserName', receiverId: 'currentUserId', receiverName: 'currentUserName', status: 'Accepted'}

addList

This event should be emitted when the user creates a list. It should emit the following 2 arguments.

currentUserId is the first argument

{title: 'List title', description: 'List description', owner: 'currentUserId', creatorName: 'currentUserName'}

editList

This event should be emitted when the user updates a list. It should emit the following 2 arguments.

currentUserId is the first argument

{id: 'list ID', title: 'List title', description: 'List description', owner: 'currentUserId', creatorName: 'currentUserName', lastModifiedBy: 'currentUserId', lastModifiedByName: 'currentUserName', lastModifiedOn: 'Current Date in ISO string format'}

deleteList

This event should be emitted when the user deletes a list. It should emit the following 2 arguments.

currentUserId is the first argument

{id: 'list ID', title: 'List title', lastModifiedByName: 'currentUserName'} is the second argument

addItem

This event should be emitted when the user adds an item into list/when user adds a subitem to the item. It should emit the following 2 arguments.

currentUserId is the first argument

{id: 'Item ID', title: 'Item title', creator: 'currentUserId', creatorName: 'currentUserName', listId: 'List id where this item added', addedOn: 'Current Date in ISO string format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'open'}

editITem

This event should be emitted when the user updates an item in list/when user updates a subitem in the item. It should emit the following 2 arguments.

currentUserId is the first argument

{id: 'Item ID', title: 'Item title', creator: 'currentUserId', creatorName: 'currentUserName', listId: 'List id where this item added', addedOn: 'Item Added Date in ISO String format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'open or done', editedOn: 'Current Date in ISO string format', editedBy: 'CurrentUserId', editedByName: 'CurrentUserName', creator: 'item creator Id', creatorName: 'item Creator Name'}

deleteItem

This event should be emitted when the user deletes an item in list/when user deletes a subitem in the item. It should emit the following 2 arguments.

currentUserId is the first argument

{id: 'Item ID', title: 'Item title', creator: 'currentUserId', creatorName: 'currentUserName', listId: 'List id where this item added', addedOn: 'Item Added Date in ISO String format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'open or done', editedOn: 'Current Date in ISO string format', editedBy: 'CurrentUserId', editedByName: 'CurrentUserName', creator: 'item creator Id', creatorName: 'item Creator Name'}

updateItemStatus

This event should be emitted when the user updates the status of an item in list/when user updates the status of a subitem for the item. It should emit the following 2 arguments.

currentUserId is the first argument

{id: 'Item ID', title: 'List title', creator: 'currentUserId', creatorName: 'currentUserName', listId: 'List id where this item added', addedOn: 'Item Added Date in ISO String format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'open or done', editedOn: 'Current Date in ISO string format', editedBy: 'CurrentUserId', editedByName: 'CurrentUserName', creator: 'item creator Id', creatorName: 'item Creator Name', completedBy: 'CurrentUserName'}

trackItemHistory

This event should be emitted for every user interaction with items/subitems. It should emit the following 2 arguments.

currentUserId is the first argument

{_id: '[Optional] item history id', title: 'List title', itemId: 'Item ID if it's item/Sub Item Id', creator: 'currentUserId', creatorName: 'currentUserName', listId: 'List id where this item added', addedOn: 'Item Added Date in ISO String format', parent: 'List ID if it's an item, Item ID if it's a subitem', itemType: 'sub or main', status: 'open or done', editedOn: 'Current Date in ISO string format', operatedById: 'CurrentUserId', operatedByName: 'CurrentUserName', operationName: 'add or edit or delete or edit-status', previliegedUsers: 'array of friends ID's', creator: 'item creator Id', creatorName: 'item Creator Name', completedBy: 'CurrentUserName', children: '[optional] if item has subitems, pass all subitems as array of objects in this main object format', oldObject:'[required for edit operation] {title: 'old title', status: 'old status', completionDate: 'old completionDate'}'}

undoLastAction

This event should be emitted when user clicks undo button or presses ctrl + z.It should emit the following 2 arguments.

currentUserId is the first argument

Item Id of last performed operation

disconnect

This event should be emitted when user logs out of the session. It won't emit any data to server.

verifyUser

This event is emitted from server side whenever the connection is made between client and server. Client needs to emit setUser whenever this event is triggered from server.

onlineUserList

This event will be emitted from server side and emits data of all users who are online currently. Client will receive the below data.

[{userId: '12233', fullName: 'Vikas Pulluri'}, {userId: '88363', fullName: 'Karthik G'}]

receivedFriendRequest

This event will be emitted from server side and emits to the user when someone sends him friend request. Client will receive the below data.

{requesterId: 'currentUserId', requesterName: 'currentUserName', receiverId: 'id of the receiver', receiverName: 'Name of the receiver', status: 'Pending'}

acceptedFriendRequest

This event will be emitted from server side and emits to the user when his friend request gets accepted. Client will receive the below data.

{requesterId: 'currentUserId', requesterName: 'currentUserName', receiverId: 'id of the receiver', receiverName: 'Name of the receiver', status: 'Accepted'}

listCreated

This event will be emitted from server side and emits list data when it gets created. Client will receive the below data.

{title: 'Some list name', description: 'list description', creatorName: 'List creator name', owner: 'List creator id'}

listEdited

This event will be emitted from server side and emits list data when it gets updated. Client will receive the below data.

{title: 'Some list name', description: 'list description', creatorName: 'List creator name', owner: 'List creator id', lastModifiedByName: 'List Editor Name'}

listDeleted

This event will be emitted from server side and emits list data when it gets deleted. Client will receive the below data.

{title: 'Some list name', description: 'list description', creatorName: 'List creator name', owner: 'List creator id', lastModifiedByName: 'List Editor Name'}

itemAdded

This event will be emitted from server side and emits item data when it gets created. Client will receive the below data.

{id: 'Item ID', title: 'Item title', creator: 'creator user id', creatorName: 'creator user name', listId: 'List id where this item added', addedOn: 'item added date in ISO string format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'open'}

itemEdited

This event will be emitted from server side and emits item data when it gets updated. Client will receive the below data.

{id: 'Item ID', title: 'Item title', creator: 'creator user id', creatorName: 'creator user name', listId: 'List id where this item added', addedOn: 'item added date in ISO string format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'open', lastModifiedByName: 'Name of the user who updated the item', lastModifiedOn: 'Date in ISO string format on which item gets updated'}

itemDeleted

This event will be emitted from server side and emits item data when it gets deleted. Client will receive the below data.

{id: 'Item ID', title: 'Item title', creator: 'creator user id', creatorName: 'creator user name', listId: 'List id where this item added', addedOn: 'item added date in ISO string format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'open', lastModifiedByName: 'Name of the user who deleted the item', lastModifiedOn: 'Date in ISO string format on which item gets deleted'}

itemStatusUpdated

This event will be emitted from server side and emits item data when it's status gets updated. Client will receive the below data.

{id: 'Item ID', title: 'Item title', creator: 'creator user id', creatorName: 'creator user name', listId: 'List id where this item added', addedOn: 'item added date in ISO string format', parent: 'List ID if it's an item, Item ID if it's a subitem', status: 'done', lastModifiedByName: 'Name of the user who updated the item', lastModifiedOn: 'Date in ISO string format on which item gets updated', completedBy: 'Username of person who marked it as complete'}
