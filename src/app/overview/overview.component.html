Application Overview

Here are the list of terms you come across frequently through out this application.

    Project:

  • This application mainly aims to track projects and it's related issues
  • You can create your own projects, add members to the project by clicking on Projects link, where you will be shown all projects available in this application and given an option to create your own
  • If you don't want to create a project, all the issues will be tagged to a General project where all users of this application can have read access to the project

    Issue:

  • You can create project related issues or general issues by clicking Create Issue
  • You are creating an issue means, you will be having all rights to modify the issue at anytime. You can assign this issue to any member of your project or to any member of this application incase of "General" project
  • If issue is tagged to project other than "General" project and project don't have any members, issue will be assigned to yourself.
  • You can track all your issues in Issues page equipped with multi filtered options.
  • You can view the issue by accessing issue description page where you can modify the issue depends upon your access rights.

    Issue Previlieges:

  • You will have full control over the issue if you are the creator(reporter) of the issue.
  • You can modify the issue only if you are a reporter or assignee of the issue
  • If you are assigned to an issue, you are only allowed to update assignee and basic issue details
  • You will have only read access to the issue if you are not a member of the project
